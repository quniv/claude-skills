# Ask — Quick Answer Mode

Answer the user's question directly and concisely. Do not modify, create,
or delete any files. Do not suggest follow-up implementation steps unless
explicitly asked.

**Input:** `$ARGUMENTS`

## Rules
- Read and research freely (files, search, codegraph, web) — but never write or edit
- Answer in the fewest words that are still complete and accurate
- If the question is ambiguous, state the assumption you're making and answer it
- No trailing offers to implement, refactor, or extend
